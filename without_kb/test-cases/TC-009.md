# TC-009: Verify selecting check-in and check-out dates

**URL(s) under test:** https://www.travala.com/
**Priority:** (not specified in source)

**Preconditions:**
- Calendar showing June 2026, with the date picker already open (i.e., continuing from TC-007's end state).

**Failure handling (default):** Stop and report on any failed CHECKPOINT, unless noted otherwise.

## Steps

1. **[ACTION]** If not continuing directly from TC-007 in this suite, perform setup: navigate to the homepage and select "Hanoi, Vietnam" as the destination to open the date picker (as in TC-007). **[ASSUMED — reason: standalone runnability.]**
2. **[ACTION]** Confirm the date picker calendar is visible; if not present after a reasonable wait, fail this step. **[ASSUMED — reason: "Element existence" checklist item.]**
3. **[ACTION]** Navigate the calendar (using its next/previous-month controls) until it displays June 2026. **[ASSUMED — reason: precondition "Calendar showing June 2026" requires navigation not stated in the source's own Test Steps.]**
4. **[ACTION — data caveat]** Note: the target dates (13–14 Jun 2026) are in the past relative to the current execution date. If the calendar UI disables/greys out past dates so that 13 or 14 June 2026 cannot be clicked, **stop and report that this test cannot be executed as written** — do not silently substitute different dates. **[FLAGGED per rule 2 — the source data is now stale; this is not something the agent should guess around.]**
5. **[ACTION]** Click the calendar cell for Saturday, 13 June 2026 (check-in date).
6. **[ACTION]** Click the calendar cell for Sunday, 14 June 2026 (check-out date).
7. **[ACTION]** Wait for the date fields and summary string to update. Condition: the check-in/check-out fields show non-placeholder text. Fallback: wait up to 5 seconds. **[ASSUMED — reason: "Post-action wait" checklist item; source's "confirm both displayed" implies a wait but doesn't define one.]**
8. **[CHECKPOINT]** Confirm the check-in field displays "13 Jun 2026" (or an unambiguous equivalent representation of Saturday 13 June 2026). Fail otherwise.
9. **[CHECKPOINT]** Confirm the check-out field displays "14 Jun 2026" (or an unambiguous equivalent representation of Sunday 14 June 2026). Fail otherwise.
10. **[CHECKPOINT]** Confirm a summary string is shown matching "Saturday 13 June 2026 — Sunday 14 June 2026 (1 night)" — dash style/spacing may vary, but the day names, dates, and "1 night" count must match exactly. Fail if any of those literal components is missing or wrong.
11. **[CHECKPOINT]** Confirm the rooms/occupancy picker opens automatically immediately after the check-out date is selected, with no additional click required. Fail if it does not open automatically.

## Notes / Out of scope
- The source's historical Actual Result mentions the rooms picker defaulting to "2 Adults / 0 Children / 1 Room (carry-over default)." This specific default is not part of the stated Expected Result (which only says "rooms picker opens"), so it is not asserted here — occupancy configuration is addressed separately in TC-010.
