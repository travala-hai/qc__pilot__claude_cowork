# TC-001: Verify homepage loads with primary search form visible

**URL(s) under test:** https://www.travala.com/
**Priority:** (not specified in source)

**Preconditions:**
- Fresh, logged-out browser session — no prior travala.com cookies, local storage, or active session.

**Failure handling (default):** Any failed CHECKPOINT below should stop the test and be reported immediately (no silent retry), unless a step says otherwise.

## Steps

1. **[ACTION]** Start from a fresh, logged-out browser session (new incognito/private context, or clear travala.com cookies and local storage). *Carried over from stated precondition, converted into an executable setup step.*
2. **[ACTION]** Navigate to `https://www.travala.com/`.
3. **[ACTION]** Wait for the homepage to finish loading. Condition: no loading spinner/skeleton is visible AND the hero section element is present in the DOM. Fallback: if no discrete loading signal is detectable, wait up to 10 seconds. **[ASSUMED — reason: source says "wait for page load to complete" but doesn't define a signal or timeout; added per "Post-navigation wait" checklist item.]**
4. **[ACTION]** If a cookie-consent banner, newsletter signup popup, or promotional modal appears, dismiss it via its close/decline control. Otherwise, skip this step. **[ASSUMED — reason: fresh/logged-out sessions commonly trigger a first-visit popup; not mentioned in source but covered by "Ambient obstacles" checklist item.]**
5. **[CHECKPOINT]** Confirm the current URL is `https://www.travala.com/` (or an equivalent locale/trailing-slash redirect of it). Fail if the URL doesn't match. **[ASSUMED assertion — reason: source didn't specify a URL check; added per "Assertion specificity" checklist item to make page-load verification concrete.]**
6. **[CHECKPOINT]** Confirm a hero section is visible and contains non-empty headline text. Fail if the hero section or its headline is missing.
7. **[CHECKPOINT]** Confirm the primary search form is visible on the page. Fail if it is not present.
8. **[CHECKPOINT]** Confirm the primary search form displays exactly these four tabs, with this exact label text: "Stays", "Flights", "Car Rental", "Activities". Fail if any of these four is missing, mislabeled, or if the tab set doesn't match. **[ASSUMED exactness — reason: source lists the four tabs but doesn't say whether other/extra tabs would be acceptable; treating the list as exhaustive per "Assertion specificity."]**

## Notes / Out of scope
- The source's historical Actual Result also mentions header elements (promo banner, AVA program, currency selector, LOG IN/REGISTER). These are not part of the stated Expected Result, so they are intentionally **not** asserted here, per the rule against adding new coverage.
