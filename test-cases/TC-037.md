# TC-037: Verify Customer Information form layout and right-side hotel summary

**URL(s) under test:** https://www.travala.com/booking?step=information
**Priority:** (not specified in source)

**Preconditions:**
- `/booking?step=information` loaded, with a valid booking session (selected hotel, dates, and package).

**Failure handling (default):** Stop and report on any failed CHECKPOINT, unless noted otherwise.

## Steps

1. **[ACTION]** Reach the Customer Information step through the standard flow: search for a hotel, open a hotel's detail page, select a room package, and click "Book Now" / "Continue" — do **not** deep-link directly to `/booking?step=information`, since the page likely requires an active booking session (hotel/dates/package) to render correctly. **[ASSUMED — reason: the source precondition states the URL as already loaded but doesn't specify how it was reached; deep-linking without session context would likely break the page, so the standard flow is substituted. If the agent cannot reach this state via prior context, it should report a blocked precondition rather than guessing.]**
2. **[ACTION]** Wait for the booking page to finish loading. Condition: the step indicator, the form (left), and the hotel summary card (right) are all rendered. Fallback: wait up to 8 seconds. **[ASSUMED — reason: "Post-navigation wait" checklist item.]**
3. **[ACTION]** If a popup, modal, or promotional overlay appears on the booking page, dismiss it via its close control; otherwise skip. **[ASSUMED — reason: "Ambient obstacles" checklist item.]**
4. **[CHECKPOINT]** Confirm the current URL matches `/booking?step=information` (or an equivalent booking-step-1 URL pattern). Fail otherwise.
5. **[CHECKPOINT]** Confirm the step indicator shows exactly three steps, recognizable as "1. Customer Information", "2. Payment Details", and "3. Booking Completed" (minor wording/numbering variation acceptable).
6. **[CHECKPOINT]** Confirm "1. Customer Information" is shown in a visually active/current state (e.g., highlighted or bolded, distinguished from the other two). Fail if not visually distinguished as active.
7. **[CHECKPOINT]** Confirm "2. Payment Details" and "3. Booking Completed" are shown in an inactive/upcoming state (not styled as active or completed). Fail if either appears active.
8. **[CHECKPOINT]** Confirm the left-hand form contains all of the following fields, each visible and interactable: First Name (required), Last Name (required), Country Code, Phone Number (required), Email (required), Retype Email (required), Additional Request (optional). Fail if any field is missing or its required/optional marking doesn't match.
9. **[CHECKPOINT]** Confirm the right-hand hotel summary card displays: hotel name, a star/rating indicator, the selected check-in/check-out dates, the selected room name, an "AVA giveback" element, and a "BEST PRICE" box. Fail if any of these is missing.
10. **[CHECKPOINT]** Confirm the price breakdown box shows three distinct line items — "Original", "Discount", and "Total" — and that Original − Discount numerically corresponds to Total (same currency/unit). Fail if any of the three line items is missing, and report the values if the arithmetic doesn't reconcile. **[Concretizes "matches expected" per rule 2.]**
11. **[CHECKPOINT]** Confirm a voucher code field/control is present on the right-hand card. Fail if absent.
