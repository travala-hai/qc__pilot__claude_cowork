# TC-010 — Verify search submit navigates to results page with correct URL parameters

- **ID:** TC-010
- **Title:** Verify search submit navigates to results page with correct URL parameters
- **URL(s) under test:** https://www.travala.com/ → https://www.travala.com/search
- **Preconditions:** Destination "Hanoi, Vietnam"; check-in `{CHECKIN}`, check-out `{CHECKOUT}` (from TC-009); 1 room / 2 adults. Run TC-007 + TC-009 as setup; ensure occupancy shows 2 Adults / 0 Children / 1 Room (adjust in the rooms picker if carry-over differs). [ASSUMED — known-starting-state guard, since carry-over can change occupancy.]

## Steps

1. [ACTION] Confirm the **SEARCH** button is visible and enabled; if not after 5 s, fail the step. [ASSUMED — element-existence guard.]
2. [ACTION] Click **SEARCH**.
3. [ACTION] Wait until the URL path is `/search` AND at least one hotel card is rendered in the results list (condition-based). Fallback: wait up to 10 s (KB §8).
4. [CHECKPOINT] URL matches the pattern (KB §4): path `/search` with query params `check_in={CHECKIN as YYYY-MM-DD}`, `check_out={CHECKOUT as YYYY-MM-DD}`, `location=<numeric id>`, `place_types=city`, `r1=2`, `session_id=<non-empty>`, `user_currency=<3-letter code>`. Param order irrelevant. Any missing param or wrong value = fail → stop and report full URL.
5. [CHECKPOINT] Results header shows "**N properties found**" with N > 0. Fail → stop and report.
6. [CHECKPOINT] First page of hotel cards rendered within **6 s** of the SEARCH click (source: "within ~6s"; measure click→first card visible). >6 s but eventually loads = fail this checkpoint, report timing, continue.
