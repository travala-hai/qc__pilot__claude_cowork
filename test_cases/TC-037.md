# TC-037 — Verify Customer Information form layout and right-side hotel summary

- **ID:** TC-037
- **Title:** Verify Customer Information (booking step 1) form layout and right-side hotel summary
- **URL(s) under test:** https://www.travala.com/booking?step=information
- **Preconditions:** `/booking?step=information` loaded. Setup [ASSUMED — source assumes the page is already open; an autonomous agent must reach it]: run TC-023 steps 1–5 to open a detail page, click the **Reserve/Book** button on any available package, and wait for the booking page URL to contain `/booking` and `step=information`. Guest checkout — no login required (KB §6). **Do not proceed past step 1 or enter payment details.**

## Steps

1. [ACTION] Wait until the booking form fields are rendered (loading indicator cleared). Fallback: 10 s (KB §8).
2. [CHECKPOINT] Step indicator shows exactly three steps: "**1. Customer Information**" in the active state, "**2. Payment Details**" and "**3. Booking Completed**" in the inactive state (KB §6). Fail → stop and report.
3. [CHECKPOINT] Left column contains exactly these form fields, with required markers (*) as listed: **First Name\***, **Last Name\***, **Country Code**, **Phone Number\***, **Email\***, **Retype Email\***, **Additional Request (Optional)**. Missing field, extra required field, or wrong required marker = fail → stop and report the diff.
4. [CHECKPOINT] Country Code is pre-filled with a non-empty default. (Value is geo-IP-dependent — e.g. +65 in Singapore — so assert presence, not a specific code; record the value. KB §6.) Fail → report, continue.
5. [CHECKPOINT] Right-side hotel summary card contains all of: hotel name (matching the property booked in setup), star rating, check-in/check-out dates (matching the dates selected in setup), room/package name, AVA giveback line, and a "**BEST PRICE**" box (KB §6). Any element missing = fail → report the missing elements, continue.
6. [CHECKPOINT] Price breakdown box shows Original price, Discount, and Total, where **Original − Discount = Total** (exact to the cent), plus a voucher-code input. Arithmetic mismatch or missing row/input = fail → stop and report the three values.
