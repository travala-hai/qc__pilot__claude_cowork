# TC-006 — Verify destination autocomplete returns relevant categorized suggestions

- **ID:** TC-006
- **Title:** Verify destination autocomplete returns relevant categorized suggestions
- **URL(s) under test:** https://www.travala.com/
- **Preconditions:** Homepage loaded (run TC-001 steps 1–4 as setup); **Stays** tab active; destination field empty and focused.
- **Known issue:** BUG-007 — irrelevant fuzzy matches ("Hanover, USA"; lowercase "hanoi, Bogotá, CO" Property) previously surfaced. This test is expected to catch it.

## Steps

1. [ACTION] Confirm the destination input on the Stays tab is visible; if not visible after 5 s, fail the step. [ASSUMED — element-existence guard.]
2. [ACTION] Click the destination input to focus it.
3. [ACTION] If the field already contains text, clear it. [ASSUMED — field hygiene; prior-session values can carry over, KB §2.]
4. [ACTION] Type `Hanoi`.
5. [ACTION] Wait for the autocomplete dropdown to populate (at least one suggestion row rendered). Fallback: wait up to 5 s (KB §8).
6. [CHECKPOINT] Dropdown contains at least one entry for Hanoi, Vietnam (entry text contains "Hanoi" and "Vietnam"). Fail → stop and report.
7. [CHECKPOINT] Every suggestion row displays a category badge whose text is one of: **City / Neighborhood / Airport / Train station / Property** (KB §3). Any row with a missing or other badge = fail.
8. [CHECKPOINT] Relevance: no suggestion unrelated to the literal query ranks above a Hanoi-Vietnam entry. Concretely: (a) all entries whose name matches "Hanoi" in Vietnam appear before any non-Vietnam entry; (b) no entry appears whose name does not contain the substring "hanoi" (case-insensitive) — e.g. "Hanover" = fail. [ASSUMED — source says "irrelevant fuzzy matches demoted or excluded"; this is the concrete interpretation. If the team intends "demoted but allowed", relax (b) to ordering-only.] Fail → report as BUG-007-class relevance failure, list the full suggestion order, continue (non-blocking for TC-007).
