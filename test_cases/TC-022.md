# TC-022 — Verify discount badge math against strikethrough vs actual price (Apricot Hotel)

- **ID:** TC-022
- **Title:** Verify discount badge percentage matches (strikethrough − actual) / strikethrough
- **URL(s) under test:** https://www.travala.com/search (Hanoi results)
- **Preconditions:** Hanoi search results loaded (run TC-010 as setup); currency set to USD (KB §7). Target card: **Apricot Hotel**.
- **Known issue:** Prior run found badge "52% OFF" vs computed 88.4% (strikethrough appears to be a multi-night total vs per-night actual). This test is expected to catch unit mismatches.

## Steps

1. [ACTION] Locate the "Apricot Hotel" card in the results list: scroll through result pages, or use the results-page hotel-name search/filter input if one is present. [ASSUMED — source assumed the card was already visible; a locate strategy is required for autonomous execution.] If not found after scanning all pages (or after 3 pages if pagination is very long), fail: "Apricot Hotel not in current inventory" — this is a test-data availability failure, not a product bug.
2. [ACTION] Confirm the card shows a discount badge; if the card has no badge, report "no badge — not applicable" and stop (pass-with-note). [ASSUMED — badge presence is a precondition of the math check.]
3. [ACTION] Read the badge percentage (format "X% OFF TODAY", KB §4) → `BADGE`.
4. [ACTION] Read the strikethrough price → `ORIG` and the actual displayed price → `ACTUAL` (both `US$ N.NN`).
5. [ACTION] Compute `CALC = (ORIG − ACTUAL) / ORIG × 100`.
6. [CHECKPOINT] `|CALC − BADGE| ≤ 1` percentage point. [ASSUMED — source implies exact match; ±1 pt tolerance allows rounding of the displayed integer badge.] Fail → report BADGE, ORIG, ACTUAL, CALC, and note whether the discrepancy is consistent with a per-night vs multi-night unit mismatch (e.g. ORIG ≈ ACTUAL × nights × expected multiplier). Stop and report.
7. [CHECKPOINT] Units consistency: ORIG and ACTUAL are the same unit — either both per-night or both total for the stay. Concretely: if the card labels one price "per night" and the other "total" (or ORIG / ACTUAL ≈ number of nights × a plausible discount), fail with unit-mismatch diagnosis. [ASSUMED — source expected result says "same-unit prices"; card labeling is the observable signal.]
