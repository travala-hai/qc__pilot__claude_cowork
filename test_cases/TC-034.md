# TC-034 — Check display of packages when applying a room-type filter

- **ID:** TC-034
- **Title:** Check display of packages when applying the Twin room-type filter
- **URL(s) under test:** Hotel detail (property) page
- **Preconditions:** User is on a property page with a populated packages list (run TC-023 steps 1–5 as setup — InterContinental Hanoi Westlake detail page). [ASSUMED — source says only "User is on Property Page"; reusing the TC-023 property makes the test self-contained. Any property offering Twin rooms works.]
- **Note:** Source steps were high-level ("Select RoomTypes → Apply Twin filter → Verify package"). Filter mechanics and pass criteria below are concrete interpretations.

## Steps

1. [ACTION] Scroll to the room types / packages section and wait until package cards are rendered. Fallback: 10 s (KB §8).
2. [ACTION] Record the total number of packages displayed and the bed-type text of each → baseline. [ASSUMED — needed to verify filtering actually changed the list.]
3. [ACTION] Confirm a room-type / bed-type filter control containing a **Twin** option is visible (KB §5); if absent after 5 s, fail: "Twin filter not available on this property" (test-data issue, not a product bug). [ASSUMED — element-existence guard.]
4. [ACTION] Apply the **Twin** filter (click the Twin option).
5. [ACTION] Wait for the packages list to re-render (list content changed or a loading indicator cleared). Fallback: 5 s. [ASSUMED — post-action wait.]
6. [CHECKPOINT] Every package now displayed has a room/bed description containing "Twin" (case-insensitive). Any package without Twin bedding = fail → stop and report the offending package names.
7. [CHECKPOINT] Filter state is reflected in the UI: the Twin filter control shows as active/selected. Fail → report, continue. [ASSUMED — observable confirmation that the filter applied.]
8. [CHECKPOINT] If the property has at least one Twin package (true for baseline if any baseline package mentioned Twin), the filtered list is non-empty. Empty list despite Twin packages existing at baseline = fail → stop and report.
